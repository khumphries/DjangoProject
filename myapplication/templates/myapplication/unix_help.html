This page contains help information about the UNIX commands that are available to manipulate SecureWitnessShare's file system

SecureWitnessShare supports basic UNIX file manipulation via the following commands:

  • mkdir
  • cd
  • rm
  • mv

More detail on how to use each of these commands is provided below. All the commands are entered into the "command" bar on the file view page in SecureWitnessShare. SecureWitnessShare includes two types of files: directories and reports. Reports are the primary piece of information in the system, and can contain uploaded files, text, or both. Reports can be stored in directories, which can contain zero, one, or many reports and/or zero, one, or many other directories. Both reports and directories can be accessed via absolute paths (using ~ for the home directory of the current user and /root/ for the root of the file system) or via relative paths from the current directory (which is listed at the top of the file view page). Relative pathing supports the ".." operator to represent the parent directory in the same way as typical UNIX shells. When doing operations on reports, use the report's name.

The mkdir command takes a single argument and creates a new directory in the current directory with that name. It does not currently support creating directories anywhere besides the current directory.

The cd command takes a single argument, which is the absolute or relative path to another directory. It changes the current directory to that directory.

The rm command can be used to remove reports or directories. To remove directories, you must supply "-r" as the first argument, and the name of the directory to be removed as the second. Note that this is recursive and will delete all the reports and directories contained within the target directory, as well. Note that removed reports cannot be recovered. Be sure you really want to remove a report before you remove it.

The mv command can be used to move reports or directories. It always takes two arguments. The first is the report or directory to be moved (the source), and the second is the location to which the report or directory should be moved. If the second argument is a folder that exists, the source is moved into that folder with the same name. If the second argument does not exist, the source is moved to the location specified and renamed to match the second argument. This is the same syntax as the standard "mv" command in UNIX shells.

Note that the * operator is not supported
